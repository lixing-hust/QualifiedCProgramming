# issues

## 1. 是否需要额外 `input/array_sum.v`
- 现有公共定义 `sum` 与 `IntArray::full` 已可直接表达题意。
- 若再创建 `.v` 只会引入冗余桥接层，增加维护成本。
- 取舍：不创建 `input/array_sum.v`。

## 2. 溢出约束写法
- 题意给出“绝对值最多 1e9，不会超出 32 位 int”。
- 为避免依赖额外常量名，contract 中直接使用整型边界字面量：
  `-2147483648 <= sum(l) <= 2147483647`。
- 取舍：优先选择工具链最稳定的字面量表达。

## 3. `n = 0` 边界
- 循环体不会执行，返回值应为 0。
- 在列表语义中，`Zlength(l) == n` 且 `n = 0` 时，`l` 为空列表，`sum(l)` 为 0，和代码行为一致。
